Workspace Services

Plugin Identifier: 20000

Summary

This plugin provides useful functions from the NSFileManager and NSWorkSpace classes on Mac OS X.

Finder

icon:=FINDER Get icon for file type (file; type)

The type of file.

type (LONGINT)

FINDER Perform operation

success:=FINDER Perform operation (source; destination; files; operation)

source (TEXT)

The full path of the source directory on which to perform the operation.

destination (TEXT)

The full path of the destination directory on which to perform the operation.

files (ARRAY)

Array of file names in the source directory on which to perform the operation.

operation (LONGINT)

The type of operation to perform.

You can use a Finder Operation constant for operation.

success (LONGINT)

1 on success.

Discussion

Application

APPLICATION Get path

path:=APPLICATION Get path (application; type)

application (TEXT)

The application name, with or without the .app extension. Alternatively the bundle identifier.

type (LONGINT)

The format in which to receive the paths.

You can use a Path Style constant for type.

path (TEXT)

The full path of the application.

Discussion

Get the full path of a named application.

APPLICATION Launch

success:=APPLICATION Launch (application; deactivate)

application (TEXT)

The application name, with or without the .app extension. Alternatively the bundle identifier.

deactivate (LONGINT)

Whether to deactivate the current application or keep active.

You can use a Launch Option constant for deactivate.

success (LONGINT)

1, on success.

Discussion

Launch a named application.

APPLICATION Launch paths

success:=APPLICATION Launch paths (application; action; files)

application (TEXT)

The application name, with or without the .app extension. Alternatively the bundle identifier.

action (LONGINT)

Flags to specify the kind of action to perform on the documents.

You can use a Launch Option constant for action.

files (ARRAY)

Array of files paths.

success (LONGINT)

1, on succes.

Discussion

Launch an application and perform actions on an array of documents.

Full Screen (QuickTime)

FULL SCREEN BEGIN

FULL SCREEN BEGIN (width; height; menu)

width (LONGINT)

The new screen width or 0 to keeo the current resolution.

height (LONGINT)

The new screen height or 0 to keeo the current resolution.

menu (LONGINT)

Optionally how to manage the menu.

You can use a Full Screen Option constant for menu.

Discussion

Start full screen mode using the deprecated QuickTime API. This is not the same as the OS X Lion feature.

FULL SCREEN END

FULL SCREEN END

Discussion

End full screen mode. Note that the plugin will automatically end full screen mode when the database is closed.
